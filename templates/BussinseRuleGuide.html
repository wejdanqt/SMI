Business Rules Guide


This Guide will help you writing you own business rules

1-File Format

Your rules files must be in json format (.json)

2-File Structure

After creating a JSON, your file must start with the key word "Rules" to include all your rules inside it later on 

{"Rules" : {}}
3-Writing the rules

You will start writing the first rule inside "Rules" node, each rule should be indicated by "Rule" keyword followed by the rule number as shown in the example, it's important that the rules numbers to be sequntially and start with number 1 

{"Rules" : {"Rule1" : [] }}

The rule should start with the attribute you want to check, its important for the attribute to be in the database otherwise the file won't be accepted 

{"Rules" : {"Rule1" : ["variable1"] }}

After writing the variable you can check it by adding the operand , the variable and the operand should be sperated by a comma. legal operands (<,>,==,<=,>=)

{"Rules" : {"Rule1" :["variable1","=="] }}

Then you write the value you want to check sperated by a comma from the operand 

{"Rules" : {"Rule1" :["variable1","==","value"] }}

If you want to add more rules you should repate the steps, the result should look like this:

{"Rules" : {"Rule1" :["variable1","==","value"],
             "Rule2" :["variable1","==","value"]}}
4-Adding Sanction List

You will start writing the first rule inside "Rules" node, each rule should be indicated by "Rule" keyword followed by the rule number as shown in the example, it's important that the rules numbers to be sequntially and start with number 1 

{"Rules" : {"Rule1" : [] }}
